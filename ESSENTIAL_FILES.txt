1. dbi/syscall-tracer.c          [80 lines]  ★☆☆☆☆
   → Learn: ptrace basics, syscall interception
   → Simplest tool, start here

2. tools/memdump.c               [70 lines]  ★☆☆☆☆
   → Learn: Memory reading, hex dump
   → Easy to understand

3. analysis/strings.c            [60 lines]  ★☆☆☆☆
   → Learn: File parsing, string extraction
   → No ptrace, just file I/O

4. dbi/dbi-framework.c          [200 lines]  ★★★☆☆
   → Learn: Breakpoints, register inspection
   → Core DBI concepts

5. parsers/pe-parser.c          [150 lines]  ★★☆☆☆
   → Learn: Binary formats, structure parsing
   → Windows PE format

6. dbi/dbi-advanced.c           [150 lines]  ★★★★☆
   → Learn: Symbol resolution, function hooking
   → Most complex DBI

7. gui/rekit-gui.c              [350 lines]  ★★★☆☆
   → Learn: GTK, GUI integration
   → Ties everything together

───────────────────────────────────────────────────────────────

KEY CONCEPTS BY FILE:

syscall-tracer.c:
  • ptrace(PTRACE_SYSCALL)
  • Reading syscall arguments from registers
  • Entry/exit detection

dbi-framework.c:
  • Setting breakpoints (0xCC instruction)
  • ptrace(PTRACE_PEEKTEXT/POKETEXT)
  • Register dumps
  • Capstone disassembly

dbi-advanced.c:
  • ELF symbol table parsing
  • Finding functions by name
  • Hook management
  • Automatic symbol resolution

pe-parser.c:
  • DOS header (MZ signature)
  • PE header structure
  • Section headers (.text, .data)
  • Entry point location

rekit-gui.c:
  • GTK widget creation
  • Button callbacks
  • Running shell commands
  • Output capture
